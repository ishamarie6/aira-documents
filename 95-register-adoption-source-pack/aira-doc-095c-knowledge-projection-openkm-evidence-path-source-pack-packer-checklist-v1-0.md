# Mandatory Practice Statement

Knowledge projection must not convert a draft into authority. Obsidian, LLM Wiki, cached indexes, summaries, and AI retrieval artifacts are derivative unless the canonical register and Tier-0 evidence repository identify them as approved and current.

# 4. Packer and Source-Pack Acceptance Criteria

Packer includes 082-094 only through an approved include rule or supplemental lane.

Duplicate document IDs, duplicate filenames, stale versions, and conflicting statuses are detected before packaging.

Generated pack manifest lists document count, source folder, generated timestamp, classification, and checksum.

Known reconciliation items remain visible rather than hidden by pack regeneration.

Obsidian and LLM Wiki projection happens after classification and source authority checks.

Retrieval results surface status and source authority so draft artifacts are not mistaken for approved standards.

# 5. AVCI Compliance Summary
| AVCI Property | Evidence |
| --- | --- |
| Attributable | Projection, packer, evidence path, and index updates have named owners and source hashes. |
| Verifiable | Render proof, checksum manifest, freshness manifest, access test, and retrieval regression prove controlled publication. |
| Classifiable | Projection and retrieval eligibility are gated by classification and access policy. |
| Improvable | Freshness failures, retrieval mismatches, stale projections, and unresolved 00D items feed the improvement backlog. |
